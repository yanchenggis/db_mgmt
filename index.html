Spatial Database Management Toolkit

Working with spatial data often involves handling large and complex databases. In many cases, opening an entire dataset to inspect its structure or contents is time‑consuming and inefficient — especially when dealing with formats like Shapefiles, GeoJSON, or nested JSON.

This toolkit streamlines the early stages of spatial data preparation by offering lightweight, command‑line scripts to profile, clean, statistically summarize, and visualize spatial datasets — all without requiring full-featured GIS platforms.

The goal is to support faster, more transparent, and reproducible workflows during exploratory analysis and data processing, addressing frequent challenges found in real‑world projects.

Overview

This toolkit includes four Python scripts for spatial data workflows:

1Data Profiler

Purpose: Inspect large datasets and profile their structure, data quality, and spatial characteristics without loading the entire file into memory.

Key Features:

  • Preview file types: CSV, Excel, JSON, Shapefile, GeoJSON, Parquet, TSV, URL
  • Summary: row count, field list, CRS, metadata
  • Field statistics and missing‑value assessment
  • Detect coordinate columns and nested structures

Usage:

python data_profiler.py <input_file_or_url> <output_report>
Supported formats: CSV, Excel (.xlsx/.xls), JSON, Shapefile (.shp), GeoJSON, GeoPackage (.gpkg), Parquet, TSV, URL
2Data Cleaner

Purpose: Clean, filter, and convert datasets—tabular or spatial—with flexible options for column selection, row/column filtering, and multi-format export.

Key Features:

  • Load from file or URL in multiple formats
  • Select specific columns
  • Drop rows with missing values; remove columns above a null‑value threshold
  • Export to CSV, Excel, JSON, Parquet, or TSV depending on output extension

Usage:

python clean_data.py <input> <output> [--columns ...] [--drop-nulls] [--null-threshold N]

Examples:

python clean_data.py data.csv cleaned.csv --columns name,age,city
python clean_data.py data.csv cleaned.xlsx --drop-nulls
python clean_data.py data.csv cleaned.json --null-threshold 30
python clean_data.py data.csv cleaned.parquet --columns id,value --drop-nulls --null-threshold 40
Supported formats: CSV, Excel (.xlsx/.xls), JSON, GeoJSON, Shapefile (.shp), GeoPackage (.gpkg), Parquet, TSV, URL
3Chart Auto Plotter

Purpose: Generate quick, interpretable visual summaries—univariate or bivariate—from datasets with minimal commands. Appropriate chart types (bar, line, scatter) are automatically selected.

Key Features:

  • Supports CSV, Excel, JSON, Parquet, TSV, or URL inputs
  • Automatically chooses appropriate chart (bar/line/scatter)
  • Mode 1: plot counts for a column; Mode 2: plot X vs Y
  • Customize top-N, chart type, output filename

Usage:

python viz_auto.py <data> [--column COL | --x XFIELD --y YFIELD] [--chart_type...] [--top_n N] [--output FILE]

Examples:

python viz_auto.py sales.csv --column product
python viz_auto.py data.csv --x year --y population --chart_type line
python viz_auto.py data.csv --column status --top_n 15 --output counts.png
Supported formats: CSV, Excel (.xlsx/.xls), JSON, Parquet, TSV, URL
4Geospatial Visualizer

Purpose: Visualize geospatial datasets as static images or interactive HTML maps. Supports basemap overlays, attribute-based styling, and export.

Key Features:

  • Static maps: Matplotlib + Contextily with scale bar
  • Interactive maps: Folium-based pan/zoom, HTML export
  • Color features by attribute field
  • Supports individual files and directories containing shapefiles
  • Reprojects CRS automatically to match basemaps or web mapping standards

Usage:

python geo_map.py <file_or_dir> [--mode static|interactive] [--basemap ...] [--color_by FIELD] [--title "Title"] [--output FILE]

Examples:

python geo_map.py data/neighborhoods.shp --mode static
python geo_map.py data/regions.geojson --mode static --color_by population --output map.png
python geo_map.py data/districts --mode interactive --title "District Map" --output districts.html
python geo_map.py data/parks.shp --mode static --basemap satellite
Supported formats: Shapefile (.shp), GeoJSON (.geojson/.json), GeoPackage (.gpkg), KML (.kml) [directory containing .shp also supported]
Static basemaps: osm, carto, dark, satellite
