Spatial Database Management Toolkit

Working with spatial data often involves handling large and complex databases. In many cases, opening an entire dataset just to inspect its structure or contents is time-consuming and inefficient — especially when dealing with formats like Shapefiles, GeoJSON, or nested JSON.


This toolkit was developed to streamline the early stages of spatial data analysis. It provides a set of lightweight, command-line scripts to quickly profile, clean, summarize, and visualize spatial data — without the need to load the entire file into memory or rely on full-featured GIS platforms.


The goal is to support faster and more efficient workflows, particularly during exploratory analysis and data preparation. These tools reflect challenges frequently encountered in real-world projects, and are intended to make spatial data handling more transparent, reproducible, and scalable.


Overview

This toolkit contains four Python scripts designed for spatial data analysis workflow:

1 Data Profiler

Purpose: Quickly analyze and profile large datasets to understand data structure, quality, and spatial characteristics without loading entire files into memory.

Key Features:

  • View different file types (CSV, Excel, JSON, Shapefile, GeoJSON, etc.)
  • Quick overview of database details (CRS, row count, contents)
  • Field analysis and statistics for categorical values
  • Identify meaningful fields vs. high null percentage fields
  • Detect coordinate columns in non-spatial files
  • Handle nested data structures (dicts, lists)

Usage:

python data_profiler.py file_path_or_url output_path
Supported formats: CSV, Excel (.xlsx, .xls), JSON, Shapefile (.shp), GeoJSON, GeoPackage (.gpkg), Parquet, TSV, URLs
2 Data Cleaner

Purpose: Clean, filter, and convert tabular or spatial data with flexible command-line options. This script helps streamline preprocessing tasks such as selecting fields, dropping incomplete records, and removing low-quality columns — all while supporting a wide range of file formats.

Key Features:

  • Supports multiple formats: CSV, Excel, JSON, GeoJSON, Shapefile, Parquet, TSV
  • Handles both local files and remote URLs
  • Select specific columns for export
  • Drop rows containing any null values
  • Remove columns with high percentages of missing data
  • Automatically exports in the format specified by the output file extension

Usage:

python clean_data.py <input_file> <output_file> [options]

Options:

  • --columns name,age,city  →  Export only the specified columns
  • --drop-nulls  →  Remove all rows with any missing values
  • --null-threshold 30  →  Remove columns with over 30% missing values

Examples:

python clean_data.py input.csv output.csv --columns name,age,city
python clean_data.py input.csv output.csv --drop-nulls
python clean_data.py input.csv output.csv --null-threshold 20
python clean_data.py input.csv output.xlsx --columns name,age --drop-nulls --null-threshold 40
Supported formats: CSV, Excel (.xlsx, .xls), JSON, Shapefile (.shp), GeoJSON, GeoPackage (.gpkg), Parquet, TSV, URLs
