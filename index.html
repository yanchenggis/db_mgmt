Spatial Database Management Toolkit

Working with spatial data often involves handling large and complex databases. In many cases, opening an entire dataset to inspect its structure or contents is time‑consuming and inefficient, especially when dealing with formats like Shapefiles, GeoJSON, or nested JSON. This toolkit streamlines the early stages of spatial data preparation by offering lightweight, command‑line scripts to profile, clean, statistically summarize, and visualize spatial datasets — all without requiring full-featured GIS platforms. The goal is to support faster, more transparent, and reproducible workflows during exploratory analysis and data processing, addressing frequent challenges found in real‑world projects.

Toolkit Overview

This toolkit includes four Python scripts for spatial data workflows:

1view_db.py

Purpose: Quickly analyze and profile large datasets to understand data structure, quality, and spatial characteristics without loading entire files into memory.

Key Features:

  • Auto-detects file type and format
  • Prints data schema, row counts, field summary
  • Field statistics and missing‑value assessment

Usage:

python view_db.py <input_file_or_url> [> <output_file>]
Supported formats: CSV, Excel, JSON, GeoJSON, Shapefile, Parquet, GPKG, URL
2clean_db.py

Purpose: Clean tabular data by selecting fields, dropping nulls, filtering columns by missing percentage, and exporting to multiple formats.

Key Features:

  • Select or drop specific columns
  • Drop rows with missing values
  • Drop columns with more than N% missing values
  • Auto format detection and output conversion

Usage:

python clean_db.py <input> <output> [--columns ...] [--drop-nulls] [--null-threshold N]
python clean_db.py data.csv cleaned.csv --columns name,age --drop-nulls --null-threshold 20
Supported formats: CSV, Excel, JSON, Parquet, TSV
3chart.py

Purpose: This tool supports both univariate and bivariate statistical visualization, allowing users to inspect frequency distributions, categorical breakdowns, and numeric comparisons using appropriate chart types.

Key Features:

  • Automatically chooses appropriate chart (bar/line/scatter)
  • Mode 1 – Column Counts: Plot frequency counts of one column
  • Mode 2 – X vs Y: Visualize the relationship between two columns

Usage:

python chart.py <input> --column <name> [--title "..."] [--top_n N] [--output file.png]
python chart.py <input> --x <col1> --y <col2> [--chart_type bar|scatter|line|auto] [--output file.png]
Supported formats: CSV, Excel, JSON, Parquet
4viz.py

Purpose: Visualize geospatial data on either a static map (image-based) or an interactive map (HTML-based) to explore, analyze, and communicate geographic information effectively. Supports vector spatial data formats with automatic CRS handling, optional attribute coloring, and export options.

Key Features:

  • Supports static maps with basemaps and scale bars
  • Supports interactive web maps with zoom, pan, and optional titles
  • Support for color-by attributes and titles
  • Automatically detects spatial file formats and reprojects to appropriate CRS

Usage:

python geo_map.py <file_or_dir> [--mode static|interactive] [--basemap ...] [--color_by FIELD] [--title "Title"] [--output FILE]
Supported formats: SHP, GeoJSON, GPKG, KML, directory (.shp)
