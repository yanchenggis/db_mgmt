Spatial Database Management Toolkit

Working with spatial data often involves handling large and complex databases. In many cases, opening an entire dataset to inspect its structure or contents is time‑consuming and inefficient — especially when dealing with formats like Shapefiles, GeoJSON, or nested JSON.

This toolkit streamlines the early stages of spatial data preparation by offering lightweight, command‑line scripts to profile, clean, statistically summarize, and visualize spatial datasets — all without requiring full-featured GIS platforms.

The goal is to support faster, more transparent, and reproducible workflows during exploratory analysis and data processing, addressing frequent challenges found in real‑world projects.

Toolkit Overview

This toolkit includes four Python scripts for spatial data workflows:

1view_db.py

Purpose: Profile a spatial or tabular dataset: structure, fields, missing values, coordinate fields, and more.

Key Features:

  • Auto-detects file type and format
  • Prints data schema, row counts, field summary
  • Handles large datasets and spatial files

Usage:

python view_db.py <input_file_or_url>
Supported formats: CSV, Excel, JSON, GeoJSON, Shapefile, Parquet, GPKG, URL
2clean_db.py

Purpose: Clean tabular data by selecting fields, dropping nulls, filtering columns by missing percentage, and exporting to multiple formats.

Key Features:

  • Select or drop specific columns
  • Drop rows with missing values
  • Drop columns with more than N% missing values
  • Auto format detection and output conversion

Usage:

python clean_db.py <input> <output> [--columns ...] [--drop-nulls] [--null-threshold N]
python clean_db.py data.csv cleaned.csv --columns name,age --drop-nulls --null-threshold 20
Supported formats: CSV, Excel, JSON, Parquet, TSV
3chart.py

Purpose: Automatically generate bar, line, or scatter plots from tabular data using intuitive CLI.

Key Features:

  • Single-column or two-column plotting
  • Smart chart-type selection or user override
  • Output to PNG image

Usage:

python chart.py <input> --column product
python chart.py data.csv --x year --y sales --chart_type line --output sales.png
Supported formats: CSV, Excel, JSON, Parquet
4viz.py

Purpose: Create static or interactive maps from geospatial datasets. Supports attribute coloring, reprojection, and basemap overlays.

Key Features:

  • Matplotlib + contextily (static maps)
  • Folium HTML export (interactive maps)
  • Support for color-by attributes and titles

Usage:

python viz.py data/ --mode interactive --output map.html
python viz.py data/parks.shp --mode static --color_by type --basemap carto
Supported formats: SHP, GeoJSON, GPKG, KML, directory (.shp)
